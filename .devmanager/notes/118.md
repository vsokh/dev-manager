# Task 118: Convert inline styles to CSS classes

## Manager notes
448 instances of style={{. Audit and convert common layout patterns (flex, padding, positioning, gaps) to CSS classes in styles.css. Only dynamic values (state-dependent colors, opacity, transforms) should remain inline.

## Exploration findings
- 465 inline style occurrences across 40 component files
- ~140 flex layout patterns (display:flex, alignItems, gap, etc.)
- ~85 spacing patterns (padding, margin with standard values like 8px, 12px, 16px)
- ~50 typography patterns (fontSize, fontWeight, lineHeight)
- ~80 layout patterns (width, height, overflow) - partially convertible
- ~65 truly dynamic styles that MUST stay inline (hover states, computed positions, progress bars)
- ~45 positioning styles that must stay inline (tooltips, dropdowns)
- Existing styles.css has good CSS variable system and some component classes

## Proposed plan
1. Add utility classes to styles.css for common patterns:
   - Flex: .flex, .flex-col, .flex-center, .flex-between, .flex-wrap, .flex-1, .shrink-0
   - Gap: .gap-xs(4px), .gap-sm(6px), .gap-md(8px), .gap-lg(12px), .gap-xl(16px)
   - Padding: .p-xs, .p-sm, .p-md, .p-lg + directional variants
   - Typography: .text-xs, .text-sm, .text-base, .text-md, .text-lg, .font-semibold, .font-bold
   - Layout: .w-full, .overflow-hidden, .overflow-auto
   - Misc: .truncate, .text-center, .text-right, .cursor-pointer
2. Convert components file by file, starting with highest-convertible ones:
   - CardForm.tsx (~14 convertible)
   - ActionButtons.tsx (~5 convertible)
   - Timeline.tsx (~8 convertible)
   - TaskCard.tsx (~6 convertible)

   - And remaining component files
3. Keep inline: all dynamic values (conditional colors, computed positions, progress bar widths, hover states)
4. Run build to verify no regressions
