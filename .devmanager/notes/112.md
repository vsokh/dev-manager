# Task 112: Extract useTemplate hook

## Manager notes
Extract template/skill loading and project scaffolding logic from useProject.ts into a dedicated useTemplate hook.

## Exploration findings
- `useProject.ts` contains template-related state and callbacks:
  - `showTemplatePicker` state (line 32)
  - Template picker flow in `connectToServer` (lines 72-76)
  - `connectWithTemplate` callback (lines 149-182)
  - `cancelTemplatePicker` callback (lines 184-187)
- These are consumed in `App.tsx` (line 37, 74-76) and passed to `ProjectPicker`
- Template logic depends on: `setStatus`, `setConnected`, `setData`, `setProjectName`, `lastWriteTime` from useSync
- fs.ts provides: `applyTemplate`, `createDefaultState`, `syncSkills`, `discoverSkillsAndAgents`, `readSkillsConfig`, `writeState`
- `ProjectTemplate` type from `templates.ts`

## Proposed plan
1. Create `src/hooks/useTemplate.ts` with:
   - `showTemplatePicker` state
   - `connectWithTemplate` callback
   - `cancelTemplatePicker` callback
   - A `triggerTemplatePicker` function for `connectToServer` to call
2. Update `useProject.ts` to:
   - Import and use `useTemplate` hook
   - Remove extracted state/callbacks
   - Call `triggerTemplatePicker()` where `setShowTemplatePicker(true)` was
3. Ensure return type from `useProject` still exposes same API
