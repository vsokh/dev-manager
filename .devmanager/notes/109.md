# Task 109: Extract folder dialog helpers

## Manager notes
server/api.js has inline PowerShell and osascript commands for native folder dialogs. Extract these platform-specific scripts into a dedicated utility module (e.g. server/dialogs.js) with a clean async interface.

## Exploration findings
- `openNativeFolderDialog()` function at lines 8-99 of server/api.js
- Handles 3 platforms: win32 (PowerShell COM dialog), darwin (osascript), linux (zenity/kdialog)
- Used only once: line 248 in the `/api/browse/native` POST handler
- Dependencies: `execFile` from 'node:child_process', `homedir`/`platform` from 'node:os'
- The function is self-contained — no dependencies on other api.js code

## Proposed plan
1. Create `server/dialogs.js` with the `openNativeFolderDialog()` function
2. Move its imports (`execFile`, `homedir`, `platform`) to the new module
3. Export `openNativeFolderDialog` as named export
4. Import it in `server/api.js` and remove the inline function + unused imports
